# Шпаргалка по Helix

В Helix действует правило **Объект -> Действие**. Сначала вы выделяете нужный кусок текста, затем применяете к нему команду. Самая полезная команда для редактирования — **`c`** (change), которая удаляет выделенное и переходит в режим вставки.

## Изменение текста (Навигация + Действие)

### Слова (Words)
* **`miwc`** — изменить всё слово целиком (где бы ни стоял курсор внутри слова).
* **`ec`** — изменить от курсора до конца текущего слова.
* **`wc`** — изменить от курсора до начала следующего слова.

### Строки (Lines)
* **`xc`** — изменить всю строку целиком (включая перенос строки). *Если нажать `x` несколько раз, выделится несколько строк.*
* **`xd`** — удалить текущую строку без перехода во вставку.
* **`vglc`** — изменить текст от курсора до конца строки (`v` - выделение, `gl` - в конец строки, `c` - изменить).
* **`vghc`** — изменить текст от курсора до начала строки (`gh` - в начало строки).

### Абзацы (Paragraphs)
* **`mipc`** — изменить текущий абзац целиком (блок текста до пустых строк).
* **`mapd`** — удалить абзац вместе с прилегающей к нему пустой строкой.

### Скобки и кавычки (Surrounds)
* **`mi"c`** — изменить текст внутри двойных кавычек `" "`.
* **`mi'c`** — изменить текст внутри одинарных кавычек `' '`.
* **`mi(c`** — изменить текст внутри круглых скобок `( )`.
* **`mi{c`** — изменить текст внутри фигурных скобок `{ }`.
* **`ma{c`** — изменить текст вместе с самими фигурными скобками (удалить скобки и их содержимое).

*Совет: Вы можете заменить действие `c` на `y` (копировать) или `d` (удалить).*

## Пользовательские настройки (из вашего config.toml)

* **`Tab`** — переход к следующему открытому файлу (буферу).
* **`Shift + Tab`** — переход к предыдущему открытому файлу.
* **`y`, `p`, `P`, `R`** — настроены на работу с **системным буфером обмена** по умолчанию. Всё, что вы копируете через `y` или вставляете через `p`, теперь шарится с браузером и другими приложениями ОС (как в Neovim `unnamedplus`).

## Навигация по файлам и интерфейсу

* **`Space + f`** — поиск файла в проекте (File picker / нечеткий поиск).
* **`Space + b`** — переключение между уже открытыми файлами (Buffer picker).
* **`Space + e`** — открыть файловый менеджер.
* **`Space + /`** — глобальный поиск текста по всему проекту.

## Полезные повседневные команды

### Комментирование кода
* **`Ctrl + c`** — закомментировать / раскомментировать текущую строку. Работает и для множества строк, если предварительно выделить их через `x`.

### Множественные курсоры (Мультикурсоры)
Мультикурсоры в Helix реализованы невероятно удобно и заменяют макросы:
* **`C` (Shift + c)** — добавить новый курсор на строку ниже.
* **`Alt + c`** — добавить новый курсор на строку выше.
* **`%`** — выделить весь файл целиком.
* **`s`** (split) — разбить выделение по регулярному выражению. Например, нажать `x` (выделить строку), затем `s`, затем ввести пробел. На каждом пробеле появится курсор!
* **`,`** (запятая) — убрать все курсоры, оставив только один главный.

### Отмена действий
* **`u`** — отменить последнее действие (Undo).
* **`U` (Shift + u)** — вернуть отмененное (Redo).

### Сохранение и выход (Настроено через Space)
Вместо ввода команд через двоеточие, вы можете использовать быстрые сочетания с пробелом:
* **`Space + w`** — сохранить текущий файл (`:w`).
* **`Space + W`** (Shift+w) — сохранить все открытые файлы (`:wa`).
* **`Space + q`** — выйти / закрыть файл (`:q`).
* **`Space + x`** — закрыть файл без сохранения (`:q!`).
* **`Space + X`** (Shift+x) — принудительно закрыть всё без сохранения.
* **`Space + Q`** (Shift+q) — сохранить всё и выйти из редактора (`:wqa`).
* **`:format`** — отформатировать текущий файл с помощью LSP.
